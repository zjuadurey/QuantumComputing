# P9: EuroSys Evidence Bundle

## Purpose

P9 is a paper-facing bundle over existing P7.5 and P8 artifacts.

It does not introduce another FailureOps mechanism. Instead, it turns the
current evidence into compact tables and figures that can be cited from a paper.

The P0-P8 documents are development milestones. P9 is the current system view:

```text
FailureOps = paired counterfactual attribution for QEC logical failure behavior
```

with evidence from:

```text
real QEC detector records
measured decoder-runtime replay
controlled proxy/runtime experiments
```

## Inputs

P9 expects these existing artifacts:

```text
data/results/p7_google_rl_qec_decoder_effect_matrix.csv
data/results/p7_5_paired_vs_unpaired_variance.csv
data/results/p7_5_rescue_induction_features.csv
data/results/p7_5_decoder_prior_interventions.csv
data/results/p8_decoder_runtime_summary.csv
```

Generate them with:

```bash
conda run -n failureops --no-capture-output python scripts/27_run_p7_sweep.py
conda run -n failureops --no-capture-output python scripts/28_run_p7_5_analysis.py
conda run -n failureops --no-capture-output python scripts/29_measure_p8_decoder_runtime.py
```

## Run P9

```bash
conda run -n failureops --no-capture-output python scripts/30_run_p9_evidence_bundle.py
```

## What P9 Clarifies

The claim table maps each paper claim to:

```text
evidence source
metric
value
interpretation
limitation
artifact to cite
```

The realness table separates:

```text
real detector-record evidence
measured decoder-runtime replay
proxy/runtime controlled experiments
```

The pairing table records which invariants are validated and where a result is
only an agreement check.

## Scope

P9 should not be read as a new version of FailureOps.

It is a consolidation layer for the current artifact. It makes the current
claim sharper:

```text
FailureOps attributes QEC logical failure behavior by paired interventions,
and the method can be evaluated on real QEC detector records.
```

P9 still does not claim:

```text
live quantum-control-stack timing
real hardware feedback latency
full FTQC scheduler attribution
threshold estimation
new decoder design
```
