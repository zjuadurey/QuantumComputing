# P7.5: Real-Data Attribution Analyses

## Purpose

P7.5 turns the P7 real-data import into a EuroSys-facing evaluation layer.

It does not claim real runtime attribution. Instead, it asks what FailureOps can
learn from real detector-event records without runtime traces:

```text
Which real QEC shot classes are rescued or induced by a decoder-pathway change?
Does paired counterfactual evaluation reduce variance?
Does decoder-pathway sensitivity scale with memory duration?
Are real detector records sensitive to decoder-prior changes?
```


## Run P7.5

Run P7 sweep first:

```bash
conda run -n failureops --no-capture-output python scripts/27_run_p7_sweep.py
```

Then run P7.5:

```bash
conda run -n failureops --no-capture-output python scripts/28_run_p7_5_analysis.py
```

## Analyses

### Effect By Cycles

Plots `paired_delta_lfr` against QEC memory duration, split by basis and control
mode. This shows whether decoder-pathway sensitivity grows as the logical memory
experiment runs longer.

### Rescued And Induced Shot Features

Summarizes detector-event features for:

```text
rescued
induced
unchanged_failure
unchanged_success
```

The features include detector-event count, early/mid/late detector counts, burst
fraction, and first-detector position.

### Paired Vs Unpaired Variance

Bootstraps the same real records in two ways:

```text
paired:
  sample paired baseline/intervened outcomes together

unpaired:
  sample baseline and intervened outcomes independently
```

This directly tests the FailureOps paired-counterfactual design choice.

### Decoder-Prior Interventions

Re-decodes the same real detector records with PyMatching under modified DEM
priors:

```text
uniform_median_probability
flatten_log_odds_50pct
sharpen_log_odds_2x
probability_floor_1e-3
probability_ceiling_1e-2
```

These are paired decoder-policy/prior interventions over the same detector
records. They are not runtime interventions.

## Scope

P7.5 strengthens the real-data story:

```text
FailureOps detects and explains intervention-sensitive failure behavior in real
QEC shot records.
```

It still does not replace the need for true decoder-runtime traces if the claim
is about queueing, deadline misses, or decoder service latency.
