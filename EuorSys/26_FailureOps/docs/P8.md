# P8: Measured Decoder Runtime Replay

## Purpose

P8 adds measured decoder-runtime evidence without requiring access to a live
quantum device.

It replays real P7 detector-event records through a local PyMatching decoder and
records wall-clock decoder runtime with `time.perf_counter_ns()`.

The P8 claim is:

```text
FailureOps can combine real QEC detector records with measured decoder-runtime
replay traces.
```

P8 does not claim live-device runtime attribution. It does not measure quantum
control hardware, live feedback delay, or real idle exposure on a processor.

## Run P8

Small smoke run:

```bash
conda run -n failureops --no-capture-output python scripts/29_measure_p8_decoder_runtime.py \
  --max-shots 200 \
  --batch-size 32 \
  --warmup-batches 1
```

Default full slice:

```bash
conda run -n failureops --no-capture-output python scripts/29_measure_p8_decoder_runtime.py
```

The default dataset is:

```text
data/raw/google_rl_qec/google_reinforcement_learning_qec/surface_code_traditional_calibration/Z/r010
```

The default decoder pathway is:

```text
correlated_matching_decoder_with_si1000_prior
```

## Outputs

```text
data/results/p8_decoder_runtime_trace.csv
data/results/p8_decoder_runtime_batches.csv
data/results/p8_decoder_runtime_summary.csv
data/results/p8_decoder_runtime_manifest.json
figures/p8_decoder_latency_vs_syndrome_weight.png
```

The trace file contains one row per shot per repeat. It includes P5a-compatible
runtime fields such as:

```text
decoder_arrival_time
decoder_start_time
decoder_finish_time
decoder_latency
decoder_service_time
decoder_deadline_miss
```

The runtime fields are measured decoder replay times. Queue wait and backlog are
zero in the base P8 run because the script measures decode service time directly.

## Measurement Semantics

P8 builds a PyMatching object from:

```text
decoding_results/<decoder_pathway>/error_model.dem
```

Then it replays:

```text
detection_events.b8
```

using `matching.decode_batch(...)`.

Each batch records:

```text
decode_wall_time_ns
per_shot_decode_time_us
throughput_shots_per_second
```

The shot-level trace assigns the measured batch service time evenly across the
shots in that batch. For true per-shot service times, run with:

```bash
--batch-size 1
```

## Scope

P8 is stronger than the earlier proxy runtime model because it measures real
decoder execution on real detector records.

P8 is still not a live runtime trace. It should be described as:

```text
measured decoder-runtime replay
```

not:

```text
live hardware feedback runtime
```
