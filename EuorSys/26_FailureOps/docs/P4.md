# P4: Verifiable Paired Counterfactual Engine

## Purpose

P4 turns the P3 experiment matrix into a semantic evaluation engine.

It does not add a new simulator, surface-code backend, real decoder runtime, or
complex scheduler. Instead, it strengthens the intervention kernel so
FailureOps results are easier to verify, reproduce, and extend.

P4 adds:

```text
intervention registry
pair ids
event-record hashes
record hashes
pairing validator
paired treatment-effect metrics
bootstrap confidence intervals
YAML config runner
experiment manifest
basic unit tests
```

## Pairing Contract

Every intervention has an explicit registry entry:

```text
name
intervention_class
allowed_changes
required_invariants
preserve_event_record
```

Runtime and policy interventions must preserve the same underlying event
record:

```text
workload_id
stress_level
seed
shot_id
detector_events
observable_flip
decoder_prediction
```

Noise interventions are allowed to transform the detector event record, but
they must preserve the same paired identity:

```text
run_id
workload_id
stress_level
seed
shot_id
```

## Outputs

P4 writes:

```text
data/results/p4_baseline_runs.csv
data/results/p4_intervened_runs.csv
data/results/p4_pairing_validation.csv
data/results/p4_paired_effects.csv
data/results/p4_manifest.json
```

Baseline rows include:

```text
event_record_hash
record_hash
```

Intervention rows include:

```text
pair_id
baseline_record_hash
intervened_record_hash
baseline_event_record_hash
intervened_event_record_hash
pairing_valid
pairing_violations
intervention_allowed_changes
intervention_required_invariants
```

## Paired Metrics

P4 reports paired treatment-effect metrics:

```text
rescued_failure_count
induced_failure_count
unchanged_failure_count
unchanged_success_count
net_rescue_count
paired_delta_lfr
net_rescue_rate
rescue_rate_among_baseline_failures
induction_rate_among_baseline_successes
bootstrap confidence intervals
```

The paired LFR delta is computed as:

```text
(induced_failure_count - rescued_failure_count) / num_valid_pairs
```

The net rescue rate is:

```text
(rescued_failure_count - induced_failure_count) / num_valid_pairs
```

## Run P4

From the repository root:

```bash
conda run -n failureops --no-capture-output python scripts/20_run_p4_from_config.py \
  --config configs/p4_default.yaml
```

Validate pairing separately:

```bash
conda run -n failureops --no-capture-output python scripts/18_validate_pairing.py \
  --input data/results/p4_intervened_runs.csv \
  --output data/results/p4_pairing_validation.csv
```

Recompute paired effects separately:

```bash
conda run -n failureops --no-capture-output python scripts/19_analyze_p4_paired_effects.py \
  --input data/results/p4_intervened_runs.csv \
  --output data/results/p4_paired_effects.csv \
  --num-bootstrap 500 \
  --bootstrap-seed 2026
```

## Tests

Run:

```bash
pytest tests
```

The tests cover:

```text
runtime interventions preserve event hashes
noise interventions may change event hashes
pairing validator catches seed mismatch
paired delta equals induced minus rescued
registry covers every P3 intervention
```

## Remaining Gaps

P4 still does not implement:

```text
real decoder runtime traces
real decoder worker execution
surface-code case studies
exact physical-event replay for noise interventions
formal statistical tests beyond bootstrap intervals
```
