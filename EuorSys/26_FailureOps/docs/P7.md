# P7: Real QEC Record Ingestion

## Purpose

P7 adds the first real-device data path for FailureOps.

It imports the Google Quantum AI reinforcement-learning QEC dataset into the
FailureOps paired-counterfactual format. P7 focuses on real QEC shot records,
not real runtime traces.

The P7 question is:

```text
For the same real detector-event record, how does logical failure behavior
change when the decoder pathway changes?
```

## Input Dataset

The expected raw dataset location is:

```text
data/raw/google_rl_qec/google_reinforcement_learning_qec/
```

Each experiment directory contains:

```text
circuit_ideal.stim
detection_events.b8
obs_flips_actual.b8
decoding_results/<decoder_pathway>/obs_flips_predicted.b8
metadata.json
```

P7 currently uses these fields:

```text
detection_events.b8
obs_flips_actual.b8
obs_flips_predicted.b8
metadata.json
```

## Run P7

Default small real-data slice:

```bash
conda run -n failureops --no-capture-output python scripts/26_run_p7_google_rl_qec.py
```

The default directory is:

```text
data/raw/google_rl_qec/google_reinforcement_learning_qec/surface_code_traditional_calibration/Z/r010
```

The default baseline decoder is:

```text
correlated_matching_decoder_with_si1000_prior
```

The default intervened decoder is:

```text
tesseract_decoder_with_si1000_prior
```

## Run P7 Sweep

Run the decoder-pathway comparison across every discovered dataset condition:

```bash
conda run -n failureops --no-capture-output python scripts/27_run_p7_sweep.py
```

Default sweep outputs:

```text
data/results/p7_google_rl_qec_decoder_effect_matrix.csv
data/results/p7_google_rl_qec_decoder_effect_aggregate.csv
```

The matrix output has one row per real-data condition:

```text
experiment_name
code_family
control_mode
basis
cycles
baseline_lfr
intervened_lfr
paired_delta_lfr
rescued_failure_count
induced_failure_count
```

The aggregate output groups the matrix by:

```text
code_family
control_mode
basis
cycles
all
```

## Pairing Semantics

P7 preserves:

```text
run_id
workload_id
stress_level
shot_id
seed
detector_events
observable_flip
```

and changes:

```text
decoder_pathway
decoder_prediction
```

The intervention is recorded as:

```text
switch_decoder_pathway
```

The paired effect reports rescued and induced logical failures when changing
decoder pathway over the same real detector-event record.

## Scope

P7 does not claim real runtime attribution because this dataset does not include
decoder queue wait, backlog, deadline miss, timeout, or latency traces.

P7 does support a real-data FailureOps claim:

```text
FailureOps can ingest real device QEC shot records and compute paired
failure-behavior changes over the same detector-event records.
```
