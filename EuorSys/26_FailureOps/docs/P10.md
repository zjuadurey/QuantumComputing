# P10: EuroSys Main Evidence Pass

## Purpose

P10 addresses the three main evidence gaps for a EuroSys-style submission:

```text
baseline comparison
real-data robustness
measured-runtime intervention closure
```

P10 is not a new FailureOps mechanism. It is an evaluation pass over P7.5 and
P8 artifacts.

## Inputs

P10 expects:

```text
data/results/p7_google_rl_qec_decoder_effect_matrix.csv
data/results/p7_5_paired_vs_unpaired_variance.csv
data/results/p7_5_rescue_induction_features.csv
data/results/p7_5_decoder_prior_interventions.csv
data/results/p8_decoder_runtime_trace.csv
```

Generate those with:

```bash
conda run -n failureops --no-capture-output python scripts/27_run_p7_sweep.py
conda run -n failureops --no-capture-output python scripts/28_run_p7_5_analysis.py
conda run -n failureops --no-capture-output python scripts/29_measure_p8_decoder_runtime.py
```

## Run P10

```bash
conda run -n failureops --no-capture-output python scripts/31_run_p10_eurosys_main_evidence.py
```

## Outputs

```text
data/results/p10_baseline_comparison.csv
data/results/p10_realdata_robustness.csv
data/results/p10_runtime_deadline_interventions.csv
data/results/p10_runtime_deadline_summary.csv
data/results/p10_eurosys_main_evidence_manifest.json
figures/p10_eurosys_main_evidence.png
```

## Baseline Comparison

P10 compares FailureOps paired sensitivity against:

```text
plain baseline LFR
static detector burden
plain aggregate LFR delta
unpaired bootstrap estimator
```

The goal is not to invent weaker baselines. The goal is to show which parts of
the evidence are only available when paired interventions are preserved:

```text
rescued and induced transitions
paired estimator variance
intervention-sensitive rankings
```

## Real-Data Robustness

P10 reports:

```text
sign consistency across real-data conditions
cycle-level sensitivity trend
cycle-rank stability across basis and control-mode groups
paired-vs-unpaired variance robustness
decoder-prior CI checks
```

These checks support robustness over the observed real-data condition matrix.
They do not claim external replication on independent QEC datasets.

## Measured-Runtime Intervention Closure

P8 measures decoder service time on real detector records. P10 turns that trace
into a paired runtime intervention by applying deadline budgets:

```text
deadline_us in {4, 5, 6, 7, 8}
```

If a measured decoder service time exceeds the budget, P10 treats the correction
as late and drops the prediction to a default no-flip correction. It then
computes paired rescued and induced failures over the same shots.

This is a measured-service-time runtime intervention closure. It is still not a
live quantum-control-stack trace.

## Scope

P10 strengthens the EuroSys evidence story but does not solve every main-track
requirement. It still does not provide:

```text
live hardware feedback timing
real control-stack queue traces
independent public QEC datasets beyond the current Google RL QEC records
production decoder scheduling
```
